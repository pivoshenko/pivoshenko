## Use Cases

This page is a standard for:
- Creating metadata for a new repository
- Auditing and improving metadata for an existing repository

## Repository Short Description

Rules:
- One line only
- Format: `<emoji> <description>`
- No period at the end
- Focus on what the repository does, not implementation details
- Use clear and searchable wording

## Repository Topics

Use at least 7 topics, sorted alphabetically, and aligned with the repository idea.

Topic selection rules:
- Prefer commonly used and discoverable topics
- Mix category, technology, and use-case topics
- Avoid duplicate meaning (for example both singular and plural variants)
- Keep topics relevant to actual repository content

## New Repository Checklist

1. Write a short description using the required format
2. Validate the description against all short description rules
3. Select at least 5 relevant topics
4. Sort topics alphabetically
5. Remove redundant or low-signal topics
6. Confirm topics reflect repository scope and target audience

## Existing Repository Audit Checklist

1. Verify short description format: `<emoji> <description>` and no trailing period
2. Check if the description still matches the current repository purpose
3. Confirm there are at least 7 topics
4. Ensure topics are alphabetically sorted
5. Replace outdated, niche, or irrelevant topics with better trending alternatives
6. Remove duplicates and near-duplicates
7. Re-check that topics align with actual code, docs, and repository positioning
